- 각 원소마다 모든 값을 순회 해야하는 경우: O(N^2)
  ex) 전체 N개 원소들에서 연속한 K개 합을 계산할 때, 가장 큰 합을 구하는 경우
      => O(N x K)
- "연속하다는 특징"을 이용해서 투 포인터로 처리: O(N)
- 2개의 포인터(커서)를 이동하면서 계산


- 가장 간단한, 무식한 방법부터 생각
  => O(N^2) 으로 시간 복잡도 초과한다면 ?
  => "연속하다는 특징"을 이용 가능한지 확인
  => "연속"하면 투 포인터 활용
- 투 포인터로 계산하는 값(합 등)의 최대값 확인
  => int 형 터지는지 확인할 것


* 투 포인터 문제 유형
 1) 포인터 2개 모두 왼쪽 or 오른쪽 중 한 쪽에서 시작
 2) 포인터 2개 각각 서로 다른 왼쪽, 오른쪽에서 시작
   - ptr1은 왼쪽에서 시작해서 오른쪽으로 이동
   - ptr2는 오른쪽에서 시작해서 왼쪽으로 이동
   => 포인터 2개가 서로 중간 지점을 향해 이동
 3) 포인터 2개가 각각 서로 다른 배열에서 이동
 4) 배열 정렬 후, 투 포인터
